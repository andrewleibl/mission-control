## Evening Session — Lead Pipeline Dashboard Concept (11:00 PM - 11:16 PM CDT)

### Problem Identified
**70% of leads are unqualified** (fail 1-3 criteria: budget, timeline, location)
- Andrew currently calls ALL leads, wasting time on tire-kickers
- Clients only see 30% that become estimates — don't see full funnel volume
- **Retention risk:** Clients cancel because they don't see leads being generated
- **Andrew's time drain:** Calling unqualified leads instead of focusing on high-intent prospects

### Solution: Lead Pipeline Dashboard
**Mission Control-style dashboard** — single-link access, no GHL login required

**Client Experience:**
- Bookmark one URL (obfuscated slug: `mission-control.com/pipeline/a7f3k9`)
- Opens to see: HOT (Andrew's calling), WARM (client's queue), COLD (archive)
- Updates every 60 seconds via polling
- Optional: 4-digit PIN for security

**Three Views:**
1. **HOT** — "Andrew is handling these now" (all 3 criteria ✓)
2. **WARM** — "Your turn to follow up" (1-2 criteria met, notes on when to call)
3. **COLD** — "Archive" (disqualified but maybe later)

**Data Flow:**
```
Lead hits GHL → Auto-text to prospect (3 questions)
              → Tagged (Hot/Warm/Cold)  
              → Dashboard polls GHL API every 60s
              → Client link shows filtered leads
```

**Card Design:**
```
┌────────────────────────────┐
│ 🔥 HOT LEAD                │
│ Mike Johnson               │
│ $3,500 budget | April      │
│ Grand Prairie              │
│                            │
│ 📞 (214) 555-0192          │

└────────────────────────────┘
```

### Offer Structure: "The Pipeline Subscription"

**30-Day Trial:**
- **Price:** $1,200 upfront
- **Guarantee:** 5 booked estimates in 30 days
- **Risk Reversal:** $100 back per missed estimate
- **Scope:** Andrew handles HOT leads (speed-to-lead, calls, books)
- **Handoff:** WARM/COLD leads auto-delivered to dashboard with full context

**Month 2+:**
- Discuss ongoing relationship direction
- Likely: $800/month ongoing for 5+ estimates + dashboard access

**Key Differentiator:**
Clients aren't buying "leads." They're buying a *decision* — Andrew filters who gets their time.

### Technical Spec

**Stack:** Next.js (Mission Control codebase)
**Security:** Obfuscated URLs + optional 4-digit PIN
**Real-time:** 60-second polling + "Last updated" indicator
**Data Source:** GHL API (read leads, filter by tags)

**Build Components:**
1. Client slug management (who sees what)
2. GHL API integration (lead ingestion)
3. Three-view dashboard (Hot/Warm/Cold)
4. Card component with lead data
