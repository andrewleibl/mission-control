### Key Patterns Learned

**Cross-Page Consistency:**
- When updating shared elements (header logo), must update every HTML file individually
- 8 pages × 2 edits each = 16 total edits for consistency
- Risk: Missing a page causes visual inconsistency when navigating

**Browser Caching:**
- Images not appearing = browser cache holding old version
- Solutions: hard refresh (Cmd+Shift+R), cache-busting (`?v=2`), Incognito mode
- Especially problematic when user sees changes on one page but not another

**Iterative Visual Refinement:**
- Start with default sizing (65px for brands, 48px for header)
- User drives: "bigger" → bump up 15-20px at a time
- Fine-tuning: "tiny bit bigger" → +5-10px
- Different assets need different sizes (Level 3 needed 165px, BaByliss only 50px)

**User-Driven Collaboration Pattern:**
- User provides assets via `file://` URLs from Downloads
- I execute technically (copy, integrate, size, test)
- Rapid back-and-forth: change → preview → feedback → iterate
- User knows the vision, I make it happen

- Page loads with "Andis" pre-checked in Brand filter
- Products filtered to show only Andis items
- URL updates dynamically if filters change
- "Clear All Filters" removes URL params

**Pattern:** Navigation links carry filter state → Collection page interprets → UI reflects → Products filter
