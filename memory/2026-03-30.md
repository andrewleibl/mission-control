## 🎯 MAJOR ACHIEVEMENTS

### 1. Mobile Product Page Fixed
**Problem:** Product images (especially wide ones like 2Spee Bundle) overflowed screen
**Root Cause:** `aspect-ratio: 1` created a 495px × 495px square box on mobile
**Solution:** 
```css
@media (max-width: 768px) {
  .product-gallery__main {
    aspect-ratio: auto !important;  /* Break the square prison */
    max-height: none !important;
  }
}
```

### 2. Collection Page Products Loading
**Problem:** Products weren't showing on collection page at all
**Root Cause:** JavaScript `allProducts` array was empty, `extractProductsFromDom()` failing silently
**Solution:** Added fallback in `applyClientSideFilters()`:
```javascript
if (allProducts.length === 0) {
  allProducts = extractProductsFromDom();
  filteredProducts = [...allProducts];
}
```

### 3. Brand Filter Auto-Apply
**Problem:** Clicking brands on homepage didn't auto-select filters on collection page
**Root Cause:** JavaScript was matching against `value` attribute instead of `data-vendor`
**Solution:** 
```javascript
// Changed from:
const vendorCheckbox = document.querySelector(`input[name="brand"][value="${vendor}"]`);
// To:
const vendorCheckbox = document.querySelector(`input[name="brand"][data-vendor="${vendorName}"]`);
```

## ❌ MISTAKES & MISSTEPS

### Mistake 1: CSS Specificity War
**What happened:** I kept adding `!important` rules that weren't taking effect
**Why:** The desktop CSS had equal specificity and loaded first
**Lesson:** Use higher-specificity selectors (like `div.parent div.child`) or move mobile CSS to end of file with `!important`

### Mistake 2: Working Theme Confusion
**What happened:** Copied entire working theme (Expanse) which broke everything
**Why:** Completely different architecture (JSON templates vs Liquid)
**Lesson:** Don't copy whole themes - extract only the needed CSS rules

### Mistake 3: JavaScript First vs CSS First
**What happened:** Tried JavaScript fixes before exhausting CSS solutions
**Why:** JS seemed "more guaranteed" but added complexity
**Lesson:** CSS is cleaner for layout issues. JS is for behavior/interaction

### Mistake 4: Not Checking Console Earlier
**What happened:** Spent hours debugging when console had clear errors
**Why:** Didn't ask user to open DevTools console
**Lesson:** Always check browser console first for JavaScript issues

## 💡 KEY LEARNINGS

### Shopify Theme Architecture
- **Server-rendered products:** Collection pages use `{% for product in collection.products %}`
- **JavaScript enhancement:** Client-side filtering needs to preserve server-rendered HTML
- **Liquid vs JSON:** Old themes use `.liquid` templates, new themes use `.json` + sections

### CSS Specificity Hierarchy
1. Inline styles (`style="..."`) - Highest
2. ID selectors (`#element`) - High
3. Class selectors (`.class`) - Medium
4. Element selectors (`div`) - Low
5. `!important` - Overrides all above

**Mobile override pattern:**
```css
@media (max-width: 768px) {
  /* Higher specificity than desktop */
  div.product-page .product-gallery__main {
    aspect-ratio: auto !important;
  }
}
```

### JavaScript Filter Pattern
**URL Parameter → Checkbox Matching:**
- URL: `?vendor=Andis`
- Checkbox: `<input data-vendor="Andis">`
- Match: `document.querySelector('input[data-vendor="Andis"]')`

**Not** `value="andis"` (kebab-case) vs `data-vendor="Andis"` (proper name)

## 🎓 PATTERNS FOR FUTURE BUILDS

### Mobile Product Image Fix Pattern
```css
/* Desktop keeps aspect ratio */
.product-gallery__main {
  aspect-ratio: 1;
  max-height: 495px;
}

/* Mobile breaks aspect ratio */
@media (max-width: 768px) {
  .product-gallery__main {
    aspect-ratio: auto !important;
    max-height: none !important;
    width: 100% !important;
  }
  .product-gallery__main img {
    width: 100% !important;
    height: auto !important;
  }
}
```

### Filter Auto-Apply Pattern
```javascript
// URL: ?vendor=Andis
const vendorName = urlParams.get('vendor'); // "Andis"
const checkbox = document.querySelector(
  `input[name="brand"][data-vendor="${vendorName}"]`
);
if (checkbox) {
  checkbox.checked = true;
  applyFilters();
}
```

### Client-Side Filter Fallback
```javascript
// If AJAX fails, use server-rendered HTML
if (allProducts.length === 0) {
  allProducts = extractProductsFromDom();
}
```
