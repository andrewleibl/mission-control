# LESSONS LEARNED — Shopify Theme Development

## Critical Patterns for Future Builds

### 1. Shopify Section Validation Errors
**Problem:** Sections throwing "is not a valid section type" or similar validation errors
**Root Causes:**
- Complex schemas with `tag`, `class`, `blocks` properties
- Blocks without proper defaults
- Schema syntax that Shopify's validator rejects

**Solution:**
- Remove `tag` and `class` from schema entirely
- Replace dynamic blocks with hardcoded HTML content
- Use simple section names (no hyphens in presets)
- Test schema JSON with `python3 -c "import json; json.loads(...)"`

**Future Workflow:**
1. Start with hardcoded HTML sections
2. Only add schema if user needs CMS editing
3. Keep schemas minimal: `name`, `settings` array, simple `presets`

---

### 2. CSS Consolidation
**Problem:** Styles not applying, marquee not animating, broken layouts
**Root Cause:** CSS split across multiple files (`homepage-sections.css`, `theme.css`, `clutch-theme.css`), some not included in layout

**Solution:**
- Consolidate ALL CSS into single `clutch-theme.css`
- Include only one CSS file in `theme.liquid`: `{{ 'clutch-theme.css' | asset_url | stylesheet_tag }}`
- Remove references to non-existent files (`base.css`)

**Future Workflow:**
1. Always combine CSS from source builds into single file
2. Use `grep` to find all CSS references before building zip
3. Verify no 404s in browser dev tools after upload

---

### 3. Header Layout Precision
**Problem:** Logo centered on page vs positioned between specific elements
**Root Cause:** Flexbox with `justify-content: space-between` and `flex: 1` creates equal distribution, not precise positioning

**Solution:**
- Use CSS Grid for header layouts: `grid-template-columns: 280px auto 1fr`
- Fixed width for left section (search)
- Auto width for center (logo)
- Flexible right section (nav)
- Position logo with `justify-self: start` for left alignment

**Pattern for Future Headers:**
```css
.header__inner {
  display: grid;
  grid-template-columns: [left-width] auto [center] 1fr;
  align-items: center;
}
.logo { justify-self: start; margin-left: 2rem; }
```

---

### 4. Section Double-Loading
**Problem:** Sections appearing twice, duplicate content
**Root Cause:** Loading in both `theme.liquid` (global) AND `index.liquid` (template)

**Solution:**
- Remove `{% section 'header' %}` from `theme.liquid` if it's in `index.liquid`
- OR: Keep in `theme.liquid` and remove from `index.liquid`
- Never have sections loaded in both places

**Check:**
```bash
grep -r "section 'header'" layout/ templates/
```

---

### 5. Asset Management
**Problem:** Images not showing (logos, product images)
**Root Causes:**
- Images in wrong folder (assets/logos/ vs assets/)
- Using wrong Liquid filter (`image_url` vs `asset_url`)
- Not copying assets into theme before zipping

**Solution:**
- Copy all images to `shopify-theme/assets/` before zipping
- Use `{{ 'filename.png' | asset_url }}` for theme assets
- Use `{{ image | image_url }}` for uploaded images
- Verify assets are in zip: `unzip -l theme.zip | grep .png`

---

### 6. Empty Sections on First Load
**Problem:** Sections appear blank until manually configured in Shopify editor
**Root Cause:** No default content or presets in schema

**Solution:**
- Hardcode all content directly in Liquid (no blocks/presets needed)
- If using presets, include default values for ALL settings
- Trade-off: Less CMS flexibility vs guaranteed content display

**Pattern:**
```liquid
<!-- Hardcoded (always works) -->
<h1>{{ section.settings.heading | default: "Fallback Text" }}</h1>

<!-- Blocks (risky if not configured) -->
{% for block in section.blocks %}
  {{ block.settings.text }}
{% endfor %}
```

## Session: Mar 29, 2026 2:20 PM — Shopify Page Templates Fixed

### Problem
Created 6 custom page templates (brands, supplies, classes, contact, faq, reviews) but they weren't showing in Shopify's template dropdown. User was seeing 404s when clicking header links.

### Root Causes Discovered
1. **Template dropdown only shows PUBLISHED theme templates** — uploading a theme as draft doesn't make its templates available until published
2. **Page templates don't need schemas** — `page.liquid` templates work without `{% schema %}` blocks (unlike section templates)
3. **URL handle conflicts** — old pages were blocking new ones (e.g., existing `/pages/brands` caused new page to become `/pages/brands-1`)
4. **Template files must be in templates/ folder** — obvious but verified

### Resolution Steps
| Step | Action |
|------|--------|
1 | Publish the theme (templates won't appear in dropdown until theme is live) |
2 | Create pages in Shopify Admin → Pages → Add page |
3 | Select template from "Theme template" dropdown |
4 | Set URL handle to simple name (e.g., `brands` not `pages/shop/brands`) |
5 | If handle says "taken", find and delete old conflicting page |
6 | Header links automatically work if they match `/pages/HANDLE` |

### Critical Insight: URL Handle Conflicts
When Shopify says "URL handle is already taken" but no page is visible:
- Check for **hidden** or **draft** pages
- Check Pages list for pages with different titles but same handle
- Old pages may have been created by previous theme installs or apps
- Solution: Either delete old page OR use different handle and update header link

### Template vs Section Schema Difference
| Template Type | Needs Schema? | Example |
|--------------|---------------|---------|
| Section | Yes | `sections/header.liquid` |
| Page Template | No | `templates/page.brands.liquid` |
| Alternative Page Template | Optional | Can have schema for theme editor display |
